# Investor OS 项目进度

更新日期：2026-07-16  
项目状态：第一版已上线并可持续使用

## 1. 项目目标

为个人美股与 BTC 投资建立一个以价值投资为核心的决策系统，重点服务以下场景：

- 只关注行业龙头、商业确定性高的公司和 BTC
- 在开单前强制写清理由、风险、估值纪律和证伪条件
- 保存交易决策，方便之后用结果检验当时的判断
- 每周过滤市场噪音，形成可执行的价值投资复盘

## 3. 已完成功能

### 投资策略与组合护栏

- 默认决策为“先不行动”
- 核心龙头股目标占比：60%–75%
- BTC 目标占比：5%–15%
- 机会现金目标占比：10%–25%
- 单一公司原则上不超过 15%
- 新仓默认分 3–4 次进入

| 分组 | 数量 | 标的 |

| 科技 | 8 | SPCX、NVDA、GOOGL、AMZN、AAPL、TSLA、META、MSFT |
| 加密货币 | 5 | BMNR、CRCL、MSTR、COIN、MARA |

`all`、`us` 等重复系统分组和空分组没有展示。关注列表中的标的不自动等同于投资建议。

### 研究档案与评分

- 评分维度：护城河、确定性、估值、资产负债、趋势
- 已建立策略档案：BRK.B、MSFT、GOOGL、AMZN、NVDA、COST、BTC
- 其他自选标的统一显示“待建立档案”，不会自动生成没有依据的评分
- 已建立档案的标的可直接带入开单理由表

### 开单理由与决策日志

- 支持买入、加仓、减仓和不行动四种决策
- 买入或加仓前必须通过五道纪律检查
- 每笔记录包括日期、标的、价格、仓位、投资理由、估值纪律、主要风险和证伪条件
- 支持复制标准化开单理由
- 支持导出 CSV 决策日志

### 云端持久化

- 交易决策保存在 Cloudflare D1 云端数据库
- 周报主观备注保存在云端数据库
- 数据可以跨电脑和手机访问，不依赖浏览器本地存储
- 数据接口位于 `app/api/trades` 和 `app/api/settings`

### 每周复盘

- 网站可以根据决策日志和观察池状态生成周报草稿
- 已配置 Codex 自动任务：每周日 21:00（Asia/Shanghai）生成价值投资周报
- 自动任务目前重点分析 BRK.B、MSFT、GOOGL、AMZN、NVDA、COST 和 BTC
- 自动任务的输出出现在 Codex 中，目前不会自动写回网站周报历史

## 5. 数据同步边界

### 已同步

- 网站代码已同步至 GitHub
- 网站已部署至 OpenAI Sites
- 交易日志和周报备注已同步至云端 D1
- 手机和其他电脑可通过私人网站访问同一份数据

### 尚未自动同步

- Longbridge 自选分组目前是代码中的静态快照；券商账户发生变化后需要再次同步并发布
- Longbridge 实际持仓、成本价、现金比例和成交记录尚未接入网站
- 网站没有实时股价、实时估值或公司财报数据源
- GitHub 只保存代码和项目文档，不保存本次 Codex 对话记录

## 6. 已知限制

1. 当前评分是个人策略基线，不是实时投资建议。
2. 只有少数核心标的完成了研究档案，其他标的仍需逐一分析。
3. 缺少真实持仓和成本数据，因此无法准确计算组合风险、盈亏和再平衡动作。
4. 周报自动任务与网站数据库尚未打通，报告不会自动归档到网站。
5. 网站尚未提供直接修改自选分组、研究评分和目标仓位的管理界面。

## 7. 下一阶段建议

优先级从高到低：

1. 接入真实持仓、成本价、现金比例和目标仓位。
2. 为 Longbridge 自选标的逐步建立研究档案和证伪条件。
3. 接入可靠的行情、财报与估值数据源，并标注数据时间。
4. 将 Longbridge 分组同步改为服务端定时更新，而不是代码快照。
5. 增加周报历史表，让自动报告可以写回网站并按周检索。
6. 增加组合集中度、行业暴露、BTC 风险预算和再平衡提醒。

## 9. 重要原则

- 关注不等于买入
- 好公司不等于好价格
- 没有安全边际就不行动
- 仓位必须先于收益想象
- 每次开单必须写清证伪条件
- 本网站用于个人研究和决策记录，不构成投资建议
